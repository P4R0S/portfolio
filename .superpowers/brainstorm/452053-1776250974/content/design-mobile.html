// mobile strategy
Input method?
Native keyboard works. Also: a softkey bar at the bottom with the 6 most useful commands. Tap a softkey = runs that command instantly. No typing required on phone.
Font size?
10px on desktop → 12px on mobile (minimum for readability). Line length auto-wraps. No horizontal scroll.
Viewport height?
Terminal fills 100dvh. Output scrolls inside the terminal container. Keyboard push handled with dvh units (not vh).
Tab completion?
Works on desktop keyboard. On mobile: softkey bar replaces it. No broken UX when tab isn't available.
Touch scrolling?
Terminal output is `-webkit-overflow-scrolling: touch`. Natural momentum scroll inside the terminal.
prefers-reduced-motion?
All typewriter animations disabled. Output appears instantly. Blinking cursor still works (respects user system pref).
// implementation — 5 phases, in order
  • 01
    Shell scaffold — TerminalEngine context + layout
    New root layout with amber palette. TerminalEngine React context (history, output buffer, running state). Full-viewport terminal container. CRT scanline overlay CSS. Fira Code font loaded.
    app/layout.tsx app/page.tsx components/terminal/TerminalEngine.tsx components/terminal/Terminal.tsx
  • 02
    Input layer — prompt, cursor, history, tab complete
    Blinking amber cursor. Typing area with focus trap. ↑/↓ command history. Tab autocomplete for command names and project filenames. Enter submits. Escape clears input.
    components/terminal/PromptInput.tsx lib/terminal/history.ts lib/terminal/autocomplete.ts
  • 03
    Command parser + output renderer
    CommandParser maps input string → command + args. OutputRenderer takes a command result and streams it line-by-line with typewriter animation. Skippable with Enter mid-output.
    lib/terminal/parser.ts components/terminal/OutputRenderer.tsx lib/terminal/typewriter.ts
  • 04
    All commands wired to real content
    Each command reads from existing content/*.ts files. neofetch → about info. ls projects/ → projects list. cat projects/X.md → project detail. skills, history, contact all implemented. `open github` opens link. `clear` wipes output.
    lib/terminal/commands/ content/projects.ts content/skills.ts content/experience.ts
  • 05
    Boot sequence + mobile + polish
    First-load boot animation (BIOS-style lines → shell prompt). Mobile softkey bar with 6 quick commands. dvh viewport fix. prefers-reduced-motion mode. `crt off` toggle. Final amber color pass.
    components/terminal/BootSequence.tsx components/terminal/SoftkeyBar.tsx app/globals.css

// risks & mitigations
SEO — no HTML content visible to crawlers?
Add a hidden <main> with aria-hidden="false" containing pre-rendered text of all content. Screen readers and bots see it. Terminal users don't.
Contact form — how does it work in a terminal?
`send-mail` command opens an inline form in the terminal output. Name → Subject → Body → y/n confirm. Submits to existing app/api/contact/ API. No page navigation.
First-time visitors who don't know CLI?
Boot sequence ends with: "Type `help` to see available commands." Help output lists everything. Softkey bar on mobile removes all friction.
Long project lists — terminal gets cluttered?
`ls projects/` shows the list. `cat projects/NAME.md` shows one project. `clear` resets. User controls the flow — no infinite scroll problems.
